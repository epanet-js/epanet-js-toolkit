Switch between the local repo build and published epanet-js@0.8.0. The local builds use JS-side OPFS persistence: EPANET runs entirely in WASM memory for speed, then the output files are written to the browser OPFS via createWritable() after each run. Check the epanet-opfs-test/ directory in DevTools → Application → Storage → OPFS. Run this page via pnpm run serve:opfs-test for cross-origin isolation (required for OPFS writes from a Worker).
